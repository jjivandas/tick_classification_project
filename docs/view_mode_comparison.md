# View-mode comparison: dorsal-only vs ventral-only vs both

The goal of this experiment was to see how BioCLIP zero-shot and SVM
fine-tuning behave when the model is shown **only dorsal** images, **only
ventral** images, or **both views averaged per specimen** (the original
pipeline). The same 5 species and the same images are used across all three
modes — the only thing that changes is which view(s) feed the classifier.

## Headline results

| Mode | BioCLIP zero-shot | SVM 1-shot | SVM 5-shot | SVM 10-shot | SVM 35-shot |
|---|---|---|---|---|---|
| both    | 59.3% | 57.9% | 78.7% | 85.6% | **92.4%** |
| dorsal  | **60.5%** | **60.3%** | 79.9% | 85.6% | 91.3% |
| ventral | 54.1% | 49.5% | 72.3% | 79.8% | 89.1% |

(All numbers are macro accuracy. SVM numbers are means over 100 Monte-Carlo
runs; 95% CIs are in `svm_summary.csv`.)

A few takeaways:

- **Dorsal slightly beats both at low shots and in BioCLIP zero-shot.** This
  is interesting — averaging the two views per specimen doesn't help when
  data is scarce; the noise from a worse ventral view drags the average down.
- **Ventral is consistently the weakest** across all shot levels. The 1-shot
  ventral macro (49.5%) is barely above chance for 5 classes (20%) — a single
  ventral example per species is not enough.
- **Both wins at high shots** (92.4% vs 91.3% vs 89.1% at 35-shot). With
  enough training data the SVM can exploit the complementary information in
  the dorsal+ventral average.

## What the three modes mean

Each mode loads a different filtered JSON and changes how a specimen's
embedding vector is built:

| Mode | Input JSON | Specimen vector |
|---|---|---|
| both    | `final_data_cleaned.json` (1442 records, 721 specimens × 2 views) | mean of dorsal + ventral embeddings |
| dorsal  | `dorsal_only_data.json` (721 records, 1 image per specimen)        | dorsal embedding only |
| ventral | `ventral_only_data.json` (721 records, 1 image per specimen)       | ventral embedding only |

The per-view JSONs are generated by `scripts/data_cleanup.py` as filtered
subsets of `final_data_cleaned.json` — same schema, just dropping the other
view. No new BioCLIP forward passes were needed because the embedding cache
(`data/processed/emb_cache/`) is keyed by image path, and the per-view JSONs
reference the same image paths.

## How to reproduce

The `VIEW_MODE` constant at the top of notebooks 02 and 03 controls
everything. Set it to `"both"`, `"dorsal"`, or `"ventral"` and Run All.

To run all three modes hands-off:

```bash
module load miniconda3/24.1.2-py310
source activate tick_env
bash scripts/run_view_modes.sh   # nohup-able; runs nb02+nb03 for dorsal, then ventral
```

The orchestrator:

1. Copies each notebook to a temp file inside `notebooks/.tmp_<name>.ipynb`
   (your originals are never modified).
2. Patches only the `VIEW_MODE = "..."` line in the relevant cell.
3. Executes the patched copy via `jupyter nbconvert --execute --inplace`.
4. Moves the executed copy + log to `logs/view_mode_runs/<timestamp>/`.

It skips `both` mode because the previous run dirs already cover it. To
include `both`, edit the `MODES` array in `scripts/run_view_modes.sh`.

After the runs land, build the comparison:

```bash
python scripts/compare_view_modes.py
# Outputs -> results/comparison/<timestamp>/
```

The script finds the latest tagged run dir per mode automatically. Re-running
it produces a new timestamped folder so old comparisons are preserved.

  `ventral_only_data.json` next to `final_data_cleaned.json`.
- `scripts/embedding_explorer.py` — image-level points, with `view` added as
  a color-by metadata field (overlay rather than separate runs).
- `notebooks/02_bioclip_inference.ipynb` — `VIEW_MODE` constant, view-aware
  data load, view-aware species-count gate, view-aware eval loop, view-tagged
  run dir.
- `notebooks/03_finetuning_svm_new.ipynb` — `VIEW_MODE` constant, view-aware
  data load, view-aware specimen-keeping gate, generic `specimen_vec` that
  averages whatever views are present (one or two), view-tagged run dir.
- `scripts/run_view_modes.sh` — orchestrator for the four notebook runs.
- `scripts/compare_view_modes.py` — combined comparison plots + tables.
